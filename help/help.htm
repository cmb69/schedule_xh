Schedule_XH – a CMSimple_XH Plugin

Schedule_XH facilitates the coordination of meetings for members (registered through Register_XH or Memberpages_XH ≥ 3.0) of your CMSimple_XH site. It is similar to Doodle's service, but as the appointment dates have no semantic meaning to Schedule_XH, it can be used for other votings as well.

Requirements

Schedule_XH is a plugin for CMSimple_XH. It requires a UTF-8 encoded version.

Installation

The installation is done as with many other CMSimple_XH plugins. See the CMSimple_XH wiki for further details.

  1. Backup the data on your server.
  2. Unzip the distribution on your computer.
  3. Upload the whole directory schedule/ to your server into CMSimple_XH's plugins directory.
  4. Set write permissions to the subdirectories css/, data/ and languages/.
  5. Switch to "Schedule" in the back-end to check if all requirements are fulfilled.

Settings

The plugin's configuration is done as with many other CMSimple_XH plugins in the website's back-end. Select "Schedule" from "Plugins".

Localization is done in "Language". You can translate the character strings to your own language, or customize them according to your needs.

The look of the Schedule_XH can be customized in "Plugin Stylesheet".

Usage

To set up a planning/voting on a CMSimple page, use the following plugin call:

{{{PLUGIN:schedule(%NAME%, %OPTION_1%, %OPTION_2%, %OPTION_N%);}}}

The parameters have the following meaning:

%NAME%
The name of the event or other voting. This is used as basename of the CSV file in which the results are stored, so it may contain alphanumeric characters and minus signs only.
%OPTION_X%
The name of the option, which will be displayed as heading of the respective table column. This is typically a date, but as it has no semantic meaning to Schedule_XH, can actually be any string. Basically you can have as many options as you like.

It is not necessary to restrict access to this page to members; if a visitor of your site is not logged in, he will just see the current results, without being able to vote. So you can even place the planner in the template:

<?php echo schedule(%NAME%, %OPTION_1%, %OPTION_2%, %OPTION_N%);?>

However, if you want to restrict the voting to a certain user group resp. access level, you have to place the planner on a page, which can only be accessed by members with the respective authorization.

Examples

To schedule a Christmas staff party:

{{{PLUGIN:schedule('christmas', 'Dec. 18th', 'Dec. 19th', 'Dec. 22nd');}}}

To set up a voting about the color of the new team shirts:

{{{PLUGIN:schedule('shirts', 'red', 'green', 'blue');}}}

Notes

  • You can have as many planners as you like; just give them different names.
  • You can place the same planner on different pages; just use the same name.
  • Your members can change their votes as often as they like.
  • You can change the options even after some members have already voted. As the votes are assigned to the name of the option (not by position), this should work reasonably well.
